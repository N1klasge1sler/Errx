# Contributing

Thanks for your interest in contributing to Error-Clarity.

## Guidelines
- Keep changes focused and minimal
- Follow existing code style and structure
- Write clear commit messages

## Issues First
Please open an issue before submitting a pull request, especially for new features or breaking changes.

## Breaking Changes
Breaking changes must be discussed and approved before implementation.

## Code of Conduct
Be respectful and constructive. Harassment or abusive behavior will not be tolerated.
